# NetForensicX Frontend Data Guide

This document outlines all the JSON output files generated by the backend pipeline (`phase2_output` and `processed` directories). Your frontend team can use these structured JSON files to populate the SOC dashboard components.

## 1. `attack_timeline.json`
**Location:** `phase2_output/<pcap_stem>_<timestamp>/attack_timeline.json`
**Purpose:** Reconstructs the entire attack story chronologically. Use this for a vertical "Timeline" or "Narrative" component in the UI.
**Structure:** Array of events sorted chronologically by `timestamp`.
```json
[
  {
    "timestamp": 11.091171,
    "action": "Malicious Payload Transfer", 
    "source": "192.168.1.4:49188",
    "destination": "192.168.1.5:445",
    "domain": "Unknown Domain",
    "protocol": "smb",
    "score": 90,
    "session_id": "C13ryZ16G7DsaAR3x5",
    "details": ["YARA Cluster [Ransomware/Destructive] (100 pts on 93df9b96)"]
  }
]
```
**Frontend Use Cases:** 
- A vertical scrolling timeline.
- Map the `action` to specific icons (e.g., `C2 Communication` = đź“ˇ, `Ransomware Activity` = đź’€, `Lateral Movement` = âžˇď¸Ź).

## 2. `host_profiles.json`
**Location:** `phase2_output/<pcap_stem>_<timestamp>/host_profiles.json`
**Purpose:** Tracks the severity and context of individual internal machines. This is for the "Top Infected Hosts" or "Patient Zero" widget.
**Structure:** A dictionary mapping internal IP addresses to their infection profiles.
```json
{
  "192.168.1.5": {
    "ip": "192.168.1.5",
    "hostname": "PCB-WIN10",
    "infection_score": 1840,
    "intel_hits": ["YARA Cluster [C2/Backdoor] (90 pts)"],
    "suricata_alerts": ["ET MALWARE Win32/Trickbot ..."],
    "files": ["93df9b96...", "fb83f2fe..."],
    "contacted_domains": ["bad-domain.com"],
    "contacted_ips": ["185.11.22.33"]
  }
}
```
**Frontend Use Cases:** 
- Sort by `infection_score` descending to create a "Top 5 Infected Hosts" leaderboard.
- Display `hostname` and `ip` together.
- Show badges indicating the number of extracted files, connected IPs, and Suricata alerts.

## 3. `incidents_correlated.json`
**Location:** `phase2_output/<pcap_stem>_<timestamp>/incidents_correlated.json`
**Purpose:** The core dataset for "Attack Chains" and visual charts. Contains all heavily correlated network sessions mapped to specific threat severity.
**Structure:**
```json
{
  "high_severity": [ ... ],
  "medium_severity": [ ... ],
  "total_high": 12,
  "total_medium": 34,
  "protocol_distribution": { "tcp": 40, "udp": 6 },
  "service_distribution": { "http": 20, "dns": 10, "smb": 5 },
  "src_port_distribution": { "Port 49188": 5, "Port 53": 10 }
}
```
Inside the `high_severity` array, each session looks like:
```json
{
  "uid": "C13ryZ...",
  "ts": 11.091171,
  "orig_h": "192.168.1.4",
  "resp_h": "185.22.33.44",
  "proto": "tcp",
  "service": "http",
  "http": [{"host": "malware-site.com", "uri": "/payload.exe", "method": "GET"}],
  "dns": [],
  "files": ["93df9b96..."],
  "suricata_alerts": [{"signature": "ET TROJAN...", "severity": 1}],
  "score": 130,
  "severity": "HIGH",
  "intel_hits": ["YARA Cluster [C2/Backdoor]..."]
}
```
**Frontend Use Cases:** 
- Render Pie/Donut charts using `service_distribution` and `protocol_distribution`.
- Display a detailed grid/table of Attack Chains, showing exactly what happened in each session.

## 4. `unified_iocs.json`
**Location:** `phase2_output/<pcap_stem>_<timestamp>/unified_iocs.json`
**Purpose:** A flat list of every single extracted Indicator of Compromise (IOC). Ideal for a global Search page or an "Indicators Explorer" table.
**Structure:**
```json
[
  {
    "file_hash": "93df9b96...",
    "mime_type": "application/x-dosexec",
    "yara_match": "HighEntropy_Dropper,T1027_Obfuscated_Files",
    "yara_clusters": ["Obfuscation/Evasion"],
    "yara_score": 50,
    "source": "zeek_files"
  },
  {
    "ip": "185.22.33.44",
    "source": "zeek_conn",
    "is_private": false,
    "vt_score": 12,
    "abuseipdb_score": 100
  }
]
```

## 5. `run_stats.json`
**Location:** `phase2_output/<pcap_stem>_<timestamp>/run_stats.json`
**Purpose:** Shows health metrics of the pipeline itself. Use this for a small "Pipeline Health" banner.
**Structure:**
```json
{
  "total_raw": 49891,
  "dropped_private": 65,
  "clean_iocs": 441,
  "yara_scanned": 322,
  "yara_hits": 211,
  "api_enriched": 0
}
```

---

## Tips for the React Team:
1. **Dynamic Data Fetching:** You can hit the API endpoint `/api/results/{pcap_name}` to automatically grab the latest `incidents_correlated.json`. You may want to expose new FastAPI endpoints to specifically serve `host_profiles.json` and `attack_timeline.json`.
2. **Timestamps:** Convert the relative `timestamp` floats into a human-readable format like `T+11.09s` in the UI to give analysts a sense of elapsed time.
